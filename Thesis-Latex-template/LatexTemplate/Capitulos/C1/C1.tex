El objetivo de esta plantilla es brindar una estructura básica para escribir documentos largos tales como tesis de grado o posgrado, reportes técnicos, informes de avances, monografías, etc. La misma no tiene como propósito explicar, enseñar o introducir a su usuario a \LaTeX{}.

Sin embargo, en este capítulo van a figurar algunos ejemplos de comandos y estructuras de los paquetes invocados en el documento principal para brindar un uso básico de los mismo en el desarrollo del documento.




\section{Acrónimos}
\label{Intro:sec:Acronimos}

En documentos científicos se hace uso de acrónimos para reducir los términos que se repiten frecuentemente y que por lo general han sido incorporados al léxico de la temática subyacente. Con el paquete \textit{acronym}, entre otras cosas, se puede definir y usar acrónimos a lo largo del documento. Además se puede genera la lista exclusiva de acrónimos generada en el informe. 
En pocas palabras, se debe definir una entrada en el listado de acrónimos (en este documento el listado está en el documento principal), y luego en el texto usar el comando \verb|\gls{xxx}| donde \verb|xxx| es el identificador del acrónimo definido en el listado.
Ejemplo de uso de acrónimo: \gls{ieee}.

La primera vez en el documento, el acrónimo es \textit{expandido} y en los usos siguientes el mismo es generado de forma abreviada. por ejemplo: \gls{ieee}.
El paquete brinda la posibilidad de \textit{reiniciar} el estado de ``ya definido'' para que vuelva a ser \textit{expandido} en cualquier momento. 
En caso de necesitar que un acrónimo se reinicie, es decir, se vuelva a definir con el primer uso, se debe utilizar  el comando  \verb|\glsreset{xxx}|, donde \verb|xxx| es el nombre del acrónimo que se desea reiniciar. En ciertas ocasiones suele ser conveniente volver a especificar qué significa el acrónimo. 
El comando \verb|\glsresetall| reinicia todos los acrónimos para lo que resta del documento.

Para más información de este paquete, se puede consultar el manual en el siguiente link \url{http://mirrors.ctan.org/macros/latex/contrib/acronym/acronym.pdf}.

\section{Algoritmos}

En cierto tipo de documentos (especialmente aquellos con algún aspecto informático), puede ser necesario especificar un algoritmo en pseudocódigo para mostrar claramente algún procedimiento en particular. En este caso se proporciona un ejemplo usando el paquete \textit{algorithm2e}.

\begin{algorithm}[h]
	\SetKwInOut{Input}{input}
	\SetKwInOut{Output}{output}
	\SetKwRepeat{Do}{do}{while}%
	\Input{\texttt{grafo} // \gls{gelmf} atribuido con todas las métricas.}
	\Input{ \texttt{umbralUt} // valor umbral de $utilidad$.}
	\Input{ \texttt{iterativo} // indica si el filtrado es iterativo (valor booleano).}
	\Output{\texttt{GELMF'} // \texttt{grafo} sin $utilidades$.}
	
	\KwData{\texttt{cantNodos} // número de nodos eliminados en cada iteración.} 
	
	\Do{\texttt{cantNodos} != 0}{ 
		\texttt{cantNodos=0}\;	
		\ForEach{\texttt{nodo} $\in$ \texttt{grafo}}{      
			\If{\texttt{nodo.obtenerIR()} $\le$  \texttt{umbralUt} \textbf{and}\\ 
				\texttt{!nodo.esManejador()} \textbf{and} \texttt{!nodo.esInicializador()}} {
				\texttt{filtrarNodo(nodo,grafo)}\;
				\texttt{ cantNodos++}\;
			}      
		}    
		\If{\texttt{iterativo}}{
			\texttt{ajustarSUDyBC(grafo)}\;
		}
		\Else{
			\texttt{canNodos=0}\;
		}
		
	}
	\texttt{eliminarComponentesAislados(grafo)}\;
	\caption{Algoritmo de reducción de información}
	\label{Intro:alg:Proceso}
\end{algorithm}
